## Колбэки и предикаты в объектах

**Что такое колбэк и предикат**

Колбэк - это роль: функция, переданная как аргумент в другую функцию

Предикат - это вид: функция, которая возвращает true или false

Колбэк может быть предикатом, а может и не быть:

```js
// колбэк-предикат (возвращает true/false)
arr.filter(val => val > 2)

// колбэк-преобразователь (возвращает значение)
arr.map(val => val * 2)
```

**Как вызывать колбэк**

Была задача: посчитать количество значений, для которых колбэк вернул true

```js
function countIf(obj, callback) {
    let result = 0
    for (let key in obj) {
        if (callback(obj[key])) result++
    }
    return result
}

console.log(countIf({a: 1, b: 2, c: 3, d: 4}, (val) => val % 2 === 0)) // 2
```

**Я часто совершал одну и ту же ошибку: сравнивал значение с функцией**

```js
if (obj[key] === callback) // неправильно
```

`obj[key]` - это число 1, 2, 3. `callback` - это функция `(val) => val % 2 === 0`. Сравнивать число с функцией бессмысленно, условие всегда false

**Правильно: вызвать колбэк и передать в него значение**

```js
if (callback(obj[key])) // правильно
```

- `obj[key]` - значение, например 2
- `callback(2)` - вызывает `(val) => val % 2 === 0` с `val = 2`, получает `true`
- Если `true` - увеличиваем счётчик

Мы передаём значение `obj[key]` на место `val` внутри колбэка

**Как это читается**

«если колбэк от значения по ключу вернул true, увеличить счётчик»

По шагам:
- `callback` - функция, которую передали
- `(obj[key])` - вызвали её, передав значение свойства
- `if (...)` - если функция вернула true, выполняем тело

**Колбэк-преобразователь и ошибка с присваиванием**

Была задача: преобразовать все значения объекта через колбэк и вернуть новый объект

```js
function mapValues(obj, callback) {
    let newObject = {}

    for (let key in obj) {
        newObject[key] = callback(obj[key])
    }

    return newObject
}

console.log(mapValues({a: 1, b: 2, c: 3}, (val) => val * 2)) // {a: 2, b: 4, c: 6}
```

**Ещё одна моя частая ошибка: заменять объект результатом колбэка**

```js
newObject = callback(obj[key]) // неправильно
```

Каждую итерацию переменная получает новое значение:

```js
newObject = 2   // был объект - стал числом 2
newObject = 4   // был числом 2 - стал числом 4
newObject = 6   // был числом 4 - стал числом 6
```

В конце остаётся 6. Объект потерян, возвращается просто число

**Нужно добавлять свойство в объект**

```js
newObject[key] = callback(obj[key]) // правильно
```

```js
newObject["a"] = 2   // {a: 2}
newObject["b"] = 4   // {a: 2, b: 4}
newObject["c"] = 6   // {a: 2, b: 4, c: 6}
```

Объект остаётся объектом, в нём появляются свойства. Это разница между `=` и `[] =`

**Как JS понимает, какой ключ подставлять**

Ключ берётся из цикла. На каждой итерации `key` - это строка с именем текущего свойства: сначала `"a"`, потом `"b"`, потом `"c"`

```js
for (let key in obj) {
    // key = "a" (первая итерация)
    // key = "b" (вторая итерация)
    // key = "c" (третья итерация)
}
```

Когда пишешь `newObject[key]`, JS смотрит в переменную `key`, видит там `"a"` и создаёт свойство с именем `"a"`. На следующей итерации `key` уже `"b"` - создаёт `"b"`

Ты сам даёшь ключ через переменную. JS просто подставляет то, что в переменной

---

**Почему `newObject[key] = obj[key]`**

Долго путался вот с этим. Часто забывал добавить `[key]` слева и долго не мог найти ошибку, не понимал, что не так

Разбор:
- Справа `obj[key]` - берём значение из исходного объекта
- Слева `newObject[key]` - кладём это значение в новый объект под тем же ключом

```js
// key = "b"
newObject["b"] = "hello"
// Теперь newObject = {b: "hello"}
```

Если слева написать просто `newObject` без `[key]` - переменная заменится целиком, а не наполнится свойствами

**Вывод**

- Колбэк нужно вызывать: `callback(значение)`, а не сравнивать с ним
- Добавлять свойства в объект: `obj[key] = значение`, а не заменять сам объект
- `=` - замена переменной, `[] =` - добавление свойства в объект
