## Создание нового объекта

Было задание: вернуть новый объект, содержащий только те свойства, у которых строковое значение длиннее n символов

```js
function filterByLength(obj, n) {
    let newObject = {}

    for (let key in obj) {
        if (typeof obj[key] === "string" && obj[key].length > n) {
            newObject[key] = obj[key]
        }
    }

    return newObject
}

console.log(filterByLength({a: "hi", b: "hello", c: "hey", d: 42}, 3)) // {b: "hello"}
```

**Что происходит и возможные вопросы**

**1) Почему `newObject[key] = obj[key]`**

Это копирование свойства из одного объекта в другой

- Справа: `obj[key]` - берём значение из исходного объекта
- Слева: `newObject[key]` - кладём это значение в новый объект под тем же ключом

Например, `key = "b"`:
```js
newObject["b"] = "hello"
// Теперь newObject = {b: "hello"}
```

**2) Что значит `newObject[key]` и разве не происходит просто перезапись нового объекта**

`newObject[key]` - это обращение к свойству нового объекта. Если свойства с таким ключом ещё нет - оно создаётся, если уже есть - перезаписывается

Перезапись здесь не страшна, потому что в этой задаче каждый ключ проходит условие только один раз. Мы не можем дважды добавить `"b"`, потому что цикл по каждому ключу проходит ровно одну итерацию

**3) Исходный объект остаётся нетронутым**

Мы нигде не меняем `obj`, только читаем из него значения. Все изменения происходят в `newObject`. Такой подход защищает исходные данные от случайных изменений (мутаций) и делает код предсказуемым

Создание нового объекта и заполнение его в цикле - будет часто встречаться в реальных задачах
