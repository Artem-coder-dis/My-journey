## Почему `obj.length` не работает, и что такое `Object.keys`

Было задание: вернуть true, если объект пустой (нет свойств)

Я написал:

```js
function isEmpty(obj) {
    return obj === ""  // ошибка: объект сравнивается со строкой
}

console.log(isEmpty({}))                  // false (неправильно, должно быть true)
console.log(isEmpty({name: "Анна"}))      // false (правильно)
```

Но `obj === ""` проверяет, является ли obj пустой строкой, а не пустым объектом

Пустой объект - это `{}`, у него нет свойств. Чтобы проверить, есть ли у объекта свойства, используй `Object.keys(obj).length === 0`

**Но что такое `Object.keys(obj)`?**

Это метод, который возвращает **массив ключей** объекта

Пример:

```js
let user = {name: "Анна", age: 25}

Object.keys(user)  // ["name", "age"]
Object.keys({})    // [] — пустой массив, объект без свойств
```

`Object` с большой буквы - это встроенный объект в JS, у которого есть полезные методы для работы с объектами. `keys` - один из них

Если массив ключей пустой (длина === 0), значит и объект пустой - в нём нет свойств. Это стандартный способ проверки пустоты объекта

**Так что же делает `Object.keys`?**

Собирает все ключи объекта и возвращает их в виде массива строк

```js
let user = {name: "Анна", age: 25}
Object.keys(user)  // ["name", "age"]
```

Идёт по объекту, находит каждый ключ, кладёт его в массив. Порядок — как были добавлены (для обычных ключей)

**Ещё одно задание:** проверить длину объекта. Я написал:

```js
return obj.length
```

Но нет, у объектов нет свойства length, как у массивов, правильно:

```js
return Object.keys(obj).length
```

`Object.keys(obj)` возвращает массив ключей, а у массива уже есть `.length`
