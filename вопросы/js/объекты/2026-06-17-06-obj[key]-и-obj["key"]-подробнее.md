## Что такое ключ, значение и свойство

До сих пор я путался, тяжело было различать "ключ", "значение", "свойство". Что мы получаем когда пишем `obj.name` или `obj[name]`

Но я разобрался на этом задании:

Нужно было все числовые значения объекта увеличить на 1. Нечисловые оставить без изменений и вернуть обновлённый объект

```js
function incrementValues(obj) {
    for (key in obj) {
        if (typeof obj[key] === "number") {
            obj[key]++
        }
    }
    
    return obj
}

console.log(incrementValues({name: "Анна", age: 25, score: 10})) // {name: 'Анна', age: 26, score: 11}
```

**`typeof`** - это оператор определения типа. Он возвращает строку с названием типа данных, то есть возвращает строку с названием типа. Не сам тип, а слово в кавычках, пример:

- `typeof 25` → `"number"` (строка)
- `typeof "Анна"` → `"string"` (строка)
- `typeof true` → `"boolean"` (строка)

С `typeof` разобрались, так что же происходит в задаче?

Сначала циклом `for...in` мы проходимся по всем свойствам, внутри условие - если тип значения это число, увеличиваем его на 1:

`obj[key]++`

то есть `obj[key]` → `obj["age"]` → `25` → `25 + 1` → `26`

- `key` - это ключ
- `obj[key]` - это значение
- `typeof obj[key]` - это тип значения

Возможно, пока всё ещё непонятно и это нормально

**Рассмотрим на более простом примере**, есть объект:

```js
{name: "Анна", age: 25}
```

- `name` / `age` - **ключ**
- `"Анна"` / `25` - **значение**
- `name: "Анна"` / `age: 25` вместе - **свойство**

То есть когда мы вызываем объект с `[]`:

- `obj[Анна]` - JS ищет переменную с именем `Анна`. Если переменная существует, подставляет её значение как ключ, если нет - ошибка

- `obj["Анна"]` - `"Анна"` в кавычках, это строка. JS ищет ключ с именем `"Анна"` как например `age` или `name`. Если в объекте есть свойство `Анна: что-то`, вернёт его значение, а если нет - `undefined`

Разница только в том, что внутри скобок: строка в кавычках - фиксированный ключ, без кавычек - имя переменной, чьё значение станет ключом
