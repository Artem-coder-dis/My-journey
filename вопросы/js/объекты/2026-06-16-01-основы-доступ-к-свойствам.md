## Почему `obj[age]` не работает, а `obj.age` - работает?

Было задание: написать функцию `isAdult(obj)`, которая возвращает `true`, если возраст в объекте >= 18

Я написал так: 

```js
function isAdult(obj) {
    if (obj[age] >= 18) return true
    return false
}

console.log(isAdult({name: "Анна", age: 25}))  // ошибка: age is not defined
```

Почему ошибка? Ведь в объекте есть свойство `age`

Потому что `obj[age]` - без кавычек внутри скобок JS воспринимает `age` как **переменную**, а не как строку `"age"`. Переменной `age` не существует, отсюда ошибка

Правильно так:

```js
function isAdult(obj) {
    if (obj.age >= 18) return true
    return false
}
```

Или так:

```js
if (obj["age"] >= 18) return true
```

**Почему `age` - это строка, если в объекте оно без кавычек?**

В объекте ключи пишутся без кавычек, но внутри JS они хранятся как строки. `{age: 25}` - `age` здесь не переменная, а имя свойства. JS автоматически превращает его в строку `"age"`

**Три формы записи доступа к свойствам**

**1. `obj.key`** - фиксированный доступ через точку.
Ищет свойство с **буквальным** именем `"key"`. Имя жёстко прописано в коде, его нельзя подменить переменной.
*Пример:* `obj.age` - свойство `"age"`

**2. `obj["key"]`** - фиксированный доступ через скобки с кавычками.
То же самое, что `obj.key`. Кавычки внутри скобок делают ключ фиксированным.
*Пример:* `obj["age"]` - свойство `"age"`, то же, что `obj.age`

**3. `obj[key]`** - динамический доступ через скобки без кавычек.
Ищет свойство с именем, которое лежит **внутри переменной** `key`. Имя не прописано в коде, оно берётся из переменной.
*Пример:* если `key = "age"`, то `obj[key]` = `obj["age"]`

**Если `obj.key` и `obj["key"]` дают одно и то же - как лучше писать?**

**1 вариант (рекомендуемый):** `obj.age` - через точку. Короче, читаемее, современный стандарт для фиксированных ключей

**2 вариант:** `obj["age"]` - через скобки с кавычками. Используй, только если ключ содержит пробелы, дефисы или приходит из переменной (тогда без кавычек - `obj[key]`)

**Фиксированный vs Динамический - в чём разница?**

- **Фиксированный ключ** - ты точно знаешь имя свойства, когда пишешь код. `user.age`, `book.title`. Имя жёстко прописано
- **Динамический ключ** - имя свойства приходит из переменной, параметра функции или вычисляется. Ты не знаешь его заранее. `obj[key]`, где `key` может быть `"age"`, `"name"` или чем угодно

Кавычки внутри скобок всё меняют: с кавычками - фиксированный ключ, без кавычек - ключ из переменной
