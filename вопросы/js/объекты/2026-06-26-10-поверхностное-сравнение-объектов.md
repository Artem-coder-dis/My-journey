## shallowEqual: поверхностное сравнение объектов

Было задание: сравнить два объекта поверхностно - вернуть true, если у них одинаковые ключи и значения (только примитивы). Вложенные объекты/массивы не сравнивать - сразу считать разными

Я написал:

```js
function shallowEqualWt(obj1, obj2) {
    for (let Wt in obj1) {
        let val1 = obj1[Wt]
        let val2 = obj2[Wt]

        if (typeof val1 === "object" && val1 !== null) return false

        if (val1 !== val2) return false
    }

    return Object.keys(obj1).length === Object.keys(obj2).length
}

console.log(shallowEqualWt({a: 1, b: "hi"}, {b: "hi", a: 1})) // true
```

Разбираемся по шагам что происходит

**Шаг 1. создание переменных**

Создаём цикл который проходится по первому объекту

Создаём первую переменную и присваиваем ей значение из первого объекта

Создаём вторую переменную и присваиваем ей значение из второго объекта по тому же ключу

Цикл перебирает все ключи первого объекта, чтобы можно было сравнить значения по каждому ключу

**Шаг 2. первое условие**

Далее первое условие и честно, тут я разбирался долго

Сначала разберёмся что такое примитив

Примитивы - это простые типы данных в JS. Они не объекты и не имеют методов

Есть 6 типов примитивов:
- number (числа)
- string (строки)
- boolean (true/false)
- null (ничего)
- undefined (не определено)
- symbol (символы, редко)

Всё остальное - объекты (включая массивы и функции). Примитивы сравниваются по значению, объекты - по ссылке

Что значит по ссылке:

Это значит, что два объекта с одинаковым содержимым - разные для JS

```js
let a = {x: 1}
let b = {x: 1}

a === b  // false - разные ссылки, хотя содержимое одинаковое
```

JS сравнивает не содержимое, а адрес в памяти. a и b лежат в разных местах памяти, поэтому они не равны

Примитивы сравниваются по значению:

```js
let x = 1
let y = 1

x === y  // true - значение одинаковое
```

Про typeof можно прочитать [тут](типы-данных-и-операторы/2026-06-24-02-typeof.md)

Условие читается так:
Если значение - объект или массив (не примитив) → false

Вот в чём подвох: typeof для массива всегда возвращает "object". Для null как ни странно тоже "object", почему так?

*потому что это старый баг (про него можно почитать в том же посте про typeof)*

Тогда возникает противоречие:
если null входит в объекты, а условие "если мы сравниваем значение и это объект (в том числе и null) И если не равно null возвращает false"

typeof val === "object" - срабатывает на объекты, массивы И null (из-за бага)
val !== null - убирает null обратно

Правильная проверка на объект (без null) выглядит так:

```js
typeof val === "object" && val !== null
```

Понять может быть непросто, это не наша логика хромает, это в языке косяк, просто нужно смириться с этим исключением

**Шаг 3. второе условие**

```js
if (val1 !== val2) return false
```

Уже понять легче, читается оно так: "Если значения не совпадают → false"
Если оба значения примитивы и равны - идём дальше. Если хоть одно не совпало - сразу false и выходим из функции

**Шаг 4. проверка длины**

```js
return Object.keys(obj1).length === Object.keys(obj2).length
```

После цикла это ещё одно условие: проверяет, что количество ключей одинаковое. Если цикл не нашёл различий, но во втором объекте больше ключей - объекты разные, и это последнее условие отловит расхождение
