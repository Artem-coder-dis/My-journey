**typeof - оператор, который говорит тип**

`typeof` - это оператор, который возвращает строку с названием типа данных. Не сам тип, а слово в кавычках.

```js
typeof 42         // "number"
typeof "hello"    // "string"
typeof true       // "boolean"
typeof undefined  // "undefined"
```

**Задача 1: найти первое числовое значение**

```js
function findFirstNumber(obj) {
    for (let key in obj) {
        if (typeof obj[key] === "number") {
            return key
        }
    }
    return null
}

console.log(findFirstNumber({a: "hi", b: 34, c: true})) // "b"
```

`typeof obj[key] === "number"` - проверяем, что значение является числом

**Как `typeof` сравнивает число со строкой**

`typeof` всегда возвращает строку. Поэтому `typeof 34 === "number"` - это сравнение двух строк: `"number" === "number"` → `true`. Ты сравниваешь не число со строкой, а строку со строкой. Сначала `typeof` превращает значение в строку типа, потом сравниваешь

**Пример для наглядности:**

```js
let val = 34

typeof val              // "number"
typeof val === "number" // true, потому что "number" === "number"
```

**Странности typeof**

Две вещи, которые надо запомнить:

```js
typeof null        // "object" - баг, которому больше 30 лет
typeof [1, 2, 3]  // "object" - массив тоже относится к объектам
```
Почему не исправляют баг?

Потому что сломается весь интернет. Миллионы сайтов написаны с расчётом, что `typeof null === "object"`. Если исправить, старый код перестанет работать

**Как проверять правильно**

Для `null` - прямая проверка:
```js
val === null
```

Для массивов - `Array.isArray`:
```js
Array.isArray(val)
```

Для объектов (исключая null) - комбинация:
```js
typeof val === "object" && val !== null
```

**Задача 2: отличить объект от примитива**

```js
function isObject(val) {
    return typeof val === "object" && val !== null
}

console.log(isObject({a: 1}))  // true
console.log(isObject([1, 2]))  // true
console.log(isObject(null))    // false
console.log(isObject(42))      // false
```

**Итог**

`typeof` - быстрый способ узнать тип. Хорош для строк, чисел, булевых. Для массивов - `Array.isArray`, для `null` - прямая проверка
