## Что такое `truthy-значения` 
Недавно решал одну интересную задачу - нужно было вернуть количество truthy-значений в объекте

```js
function countTruthy(obj) {
    let result = 0
    for (let key in obj) {
        if (obj[key]) result++
    }

    return result
}

console.log(countTruthy({a: 1, b: 0, c: "hello", d: false})) // 2 (1 и "hello")
```

**Что такое truthy-значения?**

Truthy - это значение, которое JS считает истинным в логическом контексте (в if, в условии)
Всё, что не falsy - truthy

**Falsy-значений всего 6:**

- `false`
- `0`
- `""` (пустая строка)
- `null`
- `undefined`
- `NaN`

**Всё остальное - truthy:**

- `1`, `-5`, `3.14` (любые числа, кроме `0` и `NaN`)
- `"hello"`, `" "` (любые непустые строки)
- `[]` (пустой массив)
- `{}` (пустой объект)
- `true`

**Чем truthy отличается от привычного true?**

`true` - это одно конкретное булево значение

`truthy` - это любое значение, которое ведёт себя как `true` в условии. JS сам превращает его в `true` за кулисами, как будто делает `Boolean(значение)`. То есть если написать в `if` что угодно - число, строку, массив - JS превратит это в "да" или "нет" и решит, выполнять код дальше или пропустить

```js
if (true)     // true - это true, выполняет
if (1)        // 1 - это truthy, выполняет, хотя 1 !== true
if ("hello")  // "hello" - это truthy, выполняет, хотя "hello" !== true
if ([])       // [] - это truthy, выполняет, хотя [] !== true
```

**Так что же происходит в задаче?**

Цикл перебирает ключи объекта, для каждого ключа проверяем значение:

**Что происходит в задаче**

```js
Цикл (ключ в объекте) {
    Если (значение truthy) увеличить счётчик
}
```

Цикл перебирает ключи объекта, для каждого ключа берём значение через `obj[key]`. Если значение truthy - счётчик увеличивается, если falsy - пропускаем
